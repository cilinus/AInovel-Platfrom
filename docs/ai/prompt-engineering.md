# 프롬프트 엔지니어링

## 📋 개요

장르별 최적화된 프롬프트 템플릿. 일관된 품질과 스타일 유지.
프롬프트는 Python 모듈로 관리, 장르/스타일 조합으로 동적 생성.

---

## 🏗️ 프롬프트 구조

```
┌──────────────────────────────────────────┐
│            시스템 프롬프트                  │
│  (역할 정의 + 글쓰기 규칙 + 품질 기준)      │
├──────────────────────────────────────────┤
│            장르 프롬프트                    │
│  (장르별 톤, 클리셰, 전개 패턴)            │
├──────────────────────────────────────────┤
│            컨텍스트                        │
│  (이전 줄거리 요약, 캐릭터 정보, 세계관)    │
├──────────────────────────────────────────┤
│            생성 지시                       │
│  (이번 회차 요구사항, 분량, 특수 지시)      │
└──────────────────────────────────────────┘
```

```python
1. 한국어 웹소설 문체를 사용합니다 (구어체 + 문어체 혼합)
2. 짧은 문장 위주로 리듬감 있게 작성합니다

5. 회차 마지막에 다음 회차를 궁금하게 만드는 훅(cliffhanger)을 넣습니다
6. 성인 콘텐츠, 과도한 폭력, 혐오 표현은 절대 포함하지 않습니다

- 대화문 비율: 30~50%
- 장면 전환: 회차당 2~3회
- 캐릭터 일관성: 이전 설정과 모순 없음
"""
```

## 🔧 프롬프트 조합 함수

```python
# app/prompts/templates/__init__.py

def get_novel_prompt(
    genre: str,
    title: str,
    synopsis: str,
    previous_context: str | None = None,
    episode_number: int = 1,
    style: str = "default",
) -> str:
    """장르/컨텍스트 기반 프롬프트 조합"""

    genre_prompts = {
        "ROMANCE": ROMANCE_PROMPT,
        "FANTASY": FANTASY_PROMPT,
        "MARTIAL_ARTS": MARTIAL_ARTS_PROMPT,
        "MYSTERY": MYSTERY_PROMPT,
        "SF": SF_PROMPT,
        "MODERN": MODERN_PROMPT,
    }

    genre_prompt = genre_prompts.get(genre, "")

    context_section = ""
    if previous_context:
        context_section = f"""
## 이전 줄거리 요약
{previous_context}
"""

    return f"""{SYSTEM_PROMPT}

{genre_prompt}

## 작품 정보
- 제목: {title}
- 시놉시스: {synopsis}
- 현재 회차: {episode_number}화
{context_section}

## 생성 지시
{episode_number}화를 작성해주세요.
- 분량: 3,000~5,000자 (공백 포함)
- 이전 내용과 자연스럽게 이어지도록 합니다
- 회차 마지막에 다음이 궁금해지는 장면으로 마무리합니다
- 회차 제목을 첫 줄에 포함합니다

바로 소설 본문만 출력하세요. 메타 설명이나 주석은 포함하지 마세요.
"""


def get_continuation_prompt(
    genre: str,
    title: str,
    last_episode_content: str,
    episode_number: int,
    plot_direction: str | None = None,
) -> str:
    """이어쓰기 프롬프트"""
    base = get_novel_prompt(
        genre=genre,
        title=title,
        synopsis="",
        previous_context=last_episode_content[-2000:],  # 마지막 2000자만
        episode_number=episode_number,
    )

    if plot_direction:
        base += f"\n\n## 추가 지시\n이번 회차의 방향: {plot_direction}\n"

    return base
```

## 📊 프롬프트 토큰 최적화

| 전략 | 설명 | 절감 효과 |
|------|------|----------|
| **컨텍스트 압축** | 이전 회차 전문 대신 요약본 사용 | ~60% 입력 절감 |
| **시스템 프롬프트 캐싱** | Claude의 prompt caching 활용 | ~90% 시스템 프롬프트 절감 |
| **점진적 컨텍스트** | 최근 3화만 상세, 이전은 요약 | ~40% 입력 절감 |
| **장르 템플릿 최소화** | 핵심 키워드만 포함 | ~20% 절감 |

```python
# 컨텍스트 압축 예시
async def compress_context(episodes: list[str], max_chars: int = 2000) -> str:
    """여러 회차를 요약하여 컨텍스트 압축"""
    if not episodes:
        return ""

    # 최근 1화는 전문 (마지막 2000자)
    recent = episodes[-1][-2000:] if episodes else ""

    # 나머지는 핵심 요약
    if len(episodes) > 1:
        summary_prompt = f"다음 소설 회차들을 500자 이내로 핵심만 요약해주세요:\n\n"
        for i, ep in enumerate(episodes[:-1]):
            summary_prompt += f"[{i+1}화] {ep[:500]}...\n\n"

        summary = await generate_summary(summary_prompt)
        return f"[이전 줄거리 요약]\n{summary}\n\n[직전 회차]\n{recent}"

    return recent
```
