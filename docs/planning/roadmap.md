## 🚀 MVP (Minimum Viable Product)

### 기간: 3개월

### 목표
> **"최소 기능으로 핵심 가치 검증"**

- AI 소설 생성 + 열람 기본 플로우 완성
- 사용자 피드백 수집
- PMF (Product-Market Fit) 검증

### MVP 범위

#### ✅ 포함
| 영역 | 기능 |
|------|------|
| **회원** | 이메일 가입, 소셜 로그인 (카카오), 프로필 |
| **작품** | 소설 목록, 상세, 회차 목록 |
| **열람** | 소설 뷰어 (기본), 읽은 위치 저장 |
| **AI** | 소설 자동 생성 (3개 장르), 회차 연속 생성 |
| **토큰** | 무료 토큰 지급, 토큰 소모 |
| **서재** | 최근 본 작품, 북마크 |
| **관리자** | 기본 대시보드, 작품/회원 관리 |

#### ❌ 제외 (이후 단계)
- 결제 연동
- 만화 생성
- 구독 시스템
- 댓글/소셜 기능
- 모바일 앱
- 광고

### MVP 스프린트 계획

#### Sprint 1-2 (Week 1-4): 기반 구축
```
Week 1-2: 환경 설정
├── 프로젝트 셋업 (Next.js, NestJS)
├── DB 스키마 설계
├── 인프라 구축 (AWS)
└── CI/CD 파이프라인

Week 3-4: 인증 시스템
├── 회원가입/로그인 API
├── 카카오 소셜 로그인
├── JWT 인증
└── 프로필 관리
```

#### Sprint 3-4 (Week 5-8): 핵심 기능
```
Week 5-6: 작품/회차 시스템
├── 작품 CRUD API
├── 회차 CRUD API
├── 작품 목록 페이지
└── 작품 상세 페이지

Week 7-8: AI 생성 연동
├── OpenAI API 연동
├── 장르별 프롬프트
├── 소설 생성 파이프라인
└── 품질 검수 로직
```

#### Sprint 5-6 (Week 9-12): 통합/완성
```
Week 9-10: 열람 시스템
├── 소설 뷰어 개발
├── 읽은 위치 저장
├── 토큰 시스템
└── 서재 기능

Week 11-12: 마무리
├── 관리자 페이지
├── 테스트/QA
├── 버그 수정
└── 베타 런칭
```

### MVP 마일스톤

| 주차 | 마일스톤 | 산출물 |
|------|----------|--------|
| W2 | 인프라 완료 | 개발/스테이징 환경 |
| W4 | 인증 완료 | 로그인 기능 |
| W6 | 작품 시스템 | 작품 CRUD |

| W10 | 열람 시스템 | 뷰어 완성 |
| W12 | MVP 런칭 | 베타 서비스 |
